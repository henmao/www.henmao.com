# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Project Overview

**Docusaurus 3** static site for [www.henmao.com](https://www.henmao.com) — focused on Cocos game engine development. Content is Markdown/MDX; React/TypeScript only for custom components.

## Commands

```bash
npm start          # Dev server with hot reload
npm run build      # Build production site
npm run typecheck  # TypeScript type checking
npm run serve      # Serve built site locally
npm run clear      # Clear Docusaurus cache
```

No lint or test scripts configured.

## Directory Structure

| Path | Description |
|---|---|
| `cocos/` | Cocos game engine docs; sidebar auto-generated via `sidebars.ts` |
| `blog/` | Blog posts — `YYYY-MM-DD-slug.md` |
| `notes/` | Personal notes — `devops/`, `development/`, `tools/` |
| `src/pages/` | Custom React pages (e.g. `index.tsx`) |
| `src/components/` | React components for pages or MDX |
| `static/` | Assets copied verbatim to build output |

## Configuration

- `docusaurus.config.ts` — Site metadata, plugins, navbar/footer
- `sidebars.ts` — Sidebar for `cocos/` (autogenerated)
- `tsconfig.json` — TypeScript config (strict off, targets `cocos/` and `src/`)

## Deployment

Push to `main` → GitHub Actions (`.github/workflows/deployment.yml`) → `npm install && npm run build` → SCP `/build` to `~/docker/www/www.henmao.com` on production.

## Content Conventions

- **Blog**: `YYYY-MM-DD-slug.md` with frontmatter `title`, `description`, `authors`, `tags`
- **Cocos docs**: frontmatter `id`, `title`, `sidebar_position`; `cocos/intro.md` is the landing page
- **Notes**: frontmatter `title`, `sidebar_position`; title is short (e.g. `Linux`, `Docker`, `Nginx`)
