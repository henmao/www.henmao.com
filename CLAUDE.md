## Project Overview

This is a **Docusaurus 3** static site for [www.henmao.com](https://www.henmao.com) — a documentation and blog site focused on Cocos game engine development. Content is written in Markdown/MDX; React/TypeScript is used only for custom components.

## Commands

```bash
npm start          # Start dev server with hot reload
npm run build      # Build production static site
npm run typecheck  # TypeScript type checking
npm run serve      # Serve the built site locally
npm run clear      # Clear Docusaurus cache (use when things seem stale)
```

There is no lint or test script configured.

## Architecture

**Content sources** (processed by Docusaurus at build time):
- `cocos/` — Documentation for Cocos game engine; sidebar auto-generated via `sidebars.ts`
- `blog/` — Blog posts (MDX/MD files named `YYYY-MM-DD-title.md`)
- `src/pages/` — Custom React pages (e.g., homepage `index.tsx`)
- `src/components/` — React components used in pages or MDX
- `static/` — Assets copied verbatim to build output

**Configuration:**
- `docusaurus.config.ts` — Site metadata, plugins (local search, Google Analytics G-6W7C9317F6), themes, navbar/footer
- `sidebars.ts` — Sidebar config for `cocos/` docs (set to `autogenerated`)
- `tsconfig.json` — TypeScript config (strict mode off, targets `cocos/` and `src/`)

**Deployment:**
- Push to `main` triggers GitHub Actions (`.github/workflows/deployment.yml`)
- Workflow runs `npm install && npm run build`, then SCPs `/build` to `~/docker/www/www.henmao.com` on the production server

## Content Conventions

- Blog post filenames: `YYYY-MM-DD-slug.md` with frontmatter (`title`, `description`, `authors`, `tags`)
- Docs in `cocos/` use standard Docusaurus frontmatter (`id`, `title`, `sidebar_position`)
- The `cocos/intro.md` serves as the docs landing page (sidebar position 1)
